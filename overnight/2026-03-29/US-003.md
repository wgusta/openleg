# US-003 — Incorporate Verified Demand into Municipality Targeting

**Date:** 2026-03-29  
**Story:** US-003 — Incorporate verified demand into municipality targeting  
**Status:** ✅ Implemented & merged via PR

## Summary

Extended municipality prioritisation so the verified demand signal produced by
US-001 can influence outreach targeting alongside the two existing heuristics:

| Heuristic | Source |
|---|---|
| `energy_transition_score` | Municipality profile (renewable-energy transition progress, 0–100) |
| `leg_value_gap_chf` | Estimated annual LEG savings in CHF from ElCom tariffs |
| **`demand_score`** *(new)* | Verified resident interactions from `compute_municipality_demand_signal()` (0–100) |

A new pure function, `rank_municipalities_for_outreach()`, was added to
`insights_engine.py`.  It accepts a list of municipality profile dicts and the
output of `compute_municipality_demand_signal()`, normalises each component to
a 0–100 scale, applies configurable weights, and returns municipalities sorted
descending by the composite `outreach_score`.

**Scoring formula (default weights):**

```
outreach_score =
    0.35 × energy_transition_score          (already 0-100)
  + 0.30 × min(leg_value_gap_chf / 500, 1) × 100   (normalised, cap 500 CHF)
  + 0.35 × demand_score                    (already 0-100)
```

**Key design decisions:**

| Concept | Approach |
|---|---|
| Ownership | All ranking logic lives in `insights_engine.py`; municipality.py and api_public.py unchanged |
| Pure function | `rank_municipalities_for_outreach()` is side-effect-free; accepts data as parameters, no DB calls |
| Configurable weights | Caller can pass a custom `weights` dict to override defaults |
| Demand normalisation | `demand_score` from US-001 is already 0-100; plugged in directly |
| Value-gap normalisation | CHF value capped at 500 and scaled to 0-100 |
| Documented ranking effect | `score_breakdown` field in each output entry shows each signal's weighted contribution |
